# Security Policy

## Supported Versions

The supported version is the current `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for a vulnerability.

Report security concerns through GitHub's private vulnerability reporting if it is enabled for the repository. If it is not enabled, contact the repository owner through GitHub and share only the minimum information needed to establish a private channel.

This package is a set of local Codex skills and installation scripts. Security-sensitive reports are most likely to involve installation behavior, filesystem writes, or unsafe handling of user-provided paths.
