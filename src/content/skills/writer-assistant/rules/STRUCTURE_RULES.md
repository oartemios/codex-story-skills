# STRUCTURE_RULES

## Уровни структуры

Всегда различать:

- outline
- chapter-map
- chapter-grid
- act-breakdown
- scene-plan

## Порядок перехода

1. `outline`
2. `chapter-map`
3. `chapter-grid`
4. `act-breakdown`
5. `scene-plan`

Переходить на следующий уровень только если предыдущий уже держит основную логику книги.

## Уровни

### outline

Функция:
- зафиксировать общий ход книги и основные поворотные точки

Отличие:
- не равен `chapter-map`
- работает блоками истории, а не отдельными главами

Когда переходить дальше:
- когда понятны старт, ключевые переломы и финальный выбор

Нельзя:
- расписывать функции каждой главы
- подменять outline списком сцен
- решать локальный ритм глав

### chapter-map

Функция:
- разложить книгу по главам и зафиксировать функцию каждой главы

Отличие:
- не равен `outline`
- конкретизирует главы, а не только крупные поворотные блоки
- не равен `chapter-grid`
- не считает плотность, баланс и распределение нагрузки по книге

Когда переходить дальше:
- когда у каждой главы есть функция, внешний шаг и изменение к концу

Нельзя:
- превращать chapter-map в таблицу ритма и баланса
- описывать сцены внутри каждой главы
- дублировать outline теми же формулировками без chapter-level функции

### chapter-grid

Функция:
- проверить книгу как систему глав: ритм, чередование функций, плотность конфликтов и провисания

Отличие:
- не равен `chapter-map`
- работает не по одной главе, а по соотношению глав между собой
- не равен `act-breakdown`
- не описывает драматическую задачу актов

Когда переходить дальше:
- когда chapter-map уже собран и нужно увидеть перекосы структуры целиком

Нельзя:
- переписывать в grid содержимое каждой главы в полном виде
- подменять grid актовой логикой
- уходить в scene-level детализацию

### act-breakdown

Функция:
- собрать главы в крупные драматические блоки и показать задачу каждого акта

Отличие:
- не дублирует `chapter-grid`
- работает по блокам глав, а не по каждой главе отдельно
- уточняет драматическую функцию акта, а не ритм chapter-level сетки

Когда переходить дальше:
- когда главы уже распределены и нужно проверить крупную драматическую архитектуру

Нельзя:
- повторять полный список функций всех глав
- подменять актовую разбивку chapter-grid
- строить scene-plan до устойчивой актовой логики

### scene-plan

Функция:
- разложить конкретную главу на рабочие сцены для письма

Отличие:
- работает внутри одной главы
- не заменяет chapter-level структуру книги

Когда переходить дальше:
- только после устойчивых `chapter-map` и `chapter-grid`
- если функция главы и ее изменение уже зафиксированы

Нельзя:
- появляться раньше устойчивой структуры
- компенсировать сценами неясную функцию главы
- решать проблемный outline через локальную детализацию

## Правила

- Не перескакивать уровни
- Не писать сцены без структуры
- Каждая глава должна иметь функцию
- Если структура не держится на уровне книги, не уходить в scene-level
- Если уровень дублирует соседний уровень, структура собрана неправильно

## Ошибки

- outline, замаскированный под chapter-map
- chapter-map, замаскированный под chapter-grid
- act-breakdown, который повторяет chapter-grid
- scene-plan, появившийся до устойчивой структуры
- главы без изменения состояния
- повторяющиеся функции
- отсутствие внутреннего сдвига

## Связанные шаблоны

При работе со структурой использовать:

- ../templates/outline-template.md
- ../templates/chapter-map-template.md
- ../templates/act-breakdown-template.md

`chapter-grid` как отдельный шаблон не задан.
Если нужен grid, собирать его в рабочем формате задачи без создания нового шаблона по умолчанию.

Использовать шаблоны как опорную форму, не как жесткий формат.
