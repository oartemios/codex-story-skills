# ANALYSIS_RULES

## Назначение

Этот модуль определяет, как анализировать историю.

Главный принцип:
сначала определить тип проблемы, затем делать выводы.

---

## Типы проблем

Всегда классифицировать наблюдения:

### 1. Structural issue
Проблемы структуры:
- нет функции у главы
- нет изменения состояния
- нарушена логика развития
- сцены не двигают историю

### 2. Pacing issue
Проблемы ритма:
- провисания
- слишком быстрые переходы
- повторяющиеся блоки
- перегруз информацией

### 3. Arc issue
Проблемы арок:
- персонаж не меняется
- изменение не подготовлено
- конфликт не влияет на героя
- арка не завершена

### 4. Continuity issue
Проблемы согласованности:
- противоречия фактов
- нарушение логики мира
- несостыковки между главами

### 5. Lore issue
Проблемы лора:
- правило мира не объяснено
- система противоречит себе
- лор не влияет на сюжет

### 6. Character issue
Проблемы персонажей:
- действия не мотивированы
- поведение не соответствует характеру
- нет давления на персонажа

---

## Правила анализа

- Сначала определить входной слой анализа:
  - что именно анализируется
  - какой материал доступен
  - на каком уровне идёт анализ: сцена / глава / книга / цикл
  - хватает ли материала для уверенного вывода
- Сначала определять главный тип проблемы, затем уточнять вторичные.
- Не смешивать типы проблем без необходимости
- Не делать выводы без опоры на материал
- Не исправлять текст вместо анализа
- Не навязывать одно решение — давать варианты
- Не сваливать structural, pacing, arc и continuity в одну запись, если их можно развести по причинам и последствиям
- Если проблема выглядит составной, фиксировать:
  - первичную проблему
  - вторичные эффекты
- Если проблема не доказана полностью, обозначать её как риск

---

## Входной слой

Перед списком проблем зафиксировать:

- объект анализа
- объём материала
- уровень анализа
- главный фокус проверки
- ограничения материала

Это нужно, чтобы не смешивать общую диагностику с частной проверкой ритма, арки или согласованности.

---

## Формат вывода

Каждая проблема должна содержать:

- тип проблемы
- уровень
- где проявляется
- в чем суть
- последствия
- возможные направления решения

Если есть критичный пробел в материале, отдельно фиксировать блок `Не зафиксировано`.
