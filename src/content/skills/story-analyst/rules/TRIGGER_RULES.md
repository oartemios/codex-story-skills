# TRIGGER_RULES

## Принцип

Skill анализирует существующий материал, а не создаёт новый.

Перед выбором модуля определить:
- тип задачи
- объект анализа
- уровень (сцена / глава / книга / цикл)
- объём доступного материала
- нужен ли общий обзор или прицельная проверка одного домена

---

## Общая диагностика

Использовать `ANALYSIS_RULES.md`, если задача:
- общий анализ
- поиск слабых мест
- диагностика книги или плана

Шаблон:
- `story-diagnostics-template.md`

Дополнительно подключать:
- pacing → `PACING_RULES.md`
- arc → `ARC_RULES.md`
- consistency → `CONSISTENCY_RULES.md`

Общая диагностика не должна смешивать все типы проблем в один список без первичной классификации.

---

## Ритм

Использовать `PACING_RULES.md`, если задача:
- проверить ритм
- найти провисания
- оценить плотность событий

Шаблон:
- `pacing-review-template.md`

---

## Арки

Использовать `ARC_RULES.md`, если задача:
- анализ арки персонажа
- проверка изменений героя

Шаблон:
- `arc-review-template.md`

---

## Согласованность

Использовать `CONSISTENCY_RULES.md`, если задача:
- поиск противоречий
- проверка логики

Шаблон:
- `contradiction-report-template.md`

---

## Повторные запуски

### Создавать новый review, если:
- новый объект
- новый тип анализа
- новый уровень анализа

### Обновлять существующий review, если:
- тот же объект
- тот же тип анализа
- тот же уровень
- повторная проверка после правок

### Не создавать дубликаты

Если review уже существует и актуален — обновлять его.
Если review обновляется после правок, сохранять тот же файл и повышать версию только при изменении выводов, состава проблем или их приоритета.

---

## Размещение review

- книга → `books/<book>/reviews/`
- персонаж (глобально) → `characters/reviews/`

Не смешивать review с каноном.
