# TRIGGER_RULES

## Принцип

Skill определяет, создаётся ли новый проект или пересобирается существующий.

Перед выбором модуля определить:

- что именно просит пользователь: создать / достроить / перенести / выровнять
- есть ли уже рабочая структура проекта
- есть ли действующие опорные файлы
- нужен ли план или прямое действие
- требуется ли полный разбор базы или точечная операция

---

## Новый проект

Использовать `STARTER_RULES.md`, если:
- создать проект
- создать структуру
- сделать starter pack
- собрать минимальный каркас с нуля
- достроить пустой или почти пустой репозиторий до рабочего минимума

Шаблон:
- `starter-pack-template.md`

---

## Миграция

Использовать `MIGRATION_RULES.md`, если:
- есть существующие файлы
- нужно разобрать старую базу
- нужно разделить archive / inbox
- нужно перенести материалы в новую структуру
- есть дубликаты, старые версии, смешанные папки или неясный статус файлов

Шаблон:
- `migration-plan-template.md`

---

## Структура

Использовать `LAYOUT_RULES.md`, если:
- нужно проверить структуру
- выровнять папки
- проверить, где должны лежать уже существующие сущности
- перестроить размещение без полной миграции содержимого

---

## Ограничения

- не смешивать создание и миграцию
- сначала определить режим работы
- если запрос смешанный, выбрать основной режим, а вторичный зафиксировать отдельным блоком
- не начинать миграцию под видом layout-check
- не выдавать частичную достройку за новый проект

---

## Логика выбора

### Новый проект

Выбирать, если:

- рабочей базы ещё нет
- опорные файлы отсутствуют или почти отсутствуют
- задача состоит в создании стартового каркаса

### Миграция

Выбирать, если:

- материал уже есть, но лежит хаотично
- требуется разбор наследия
- нужно решить судьбу старых файлов и версий

### Layout

Выбирать, если:

- основная структура уже существует
- задача не в переоценке каноничности материалов, а в их правильном размещении
- нужен структурный аудит или точечное выравнивание
