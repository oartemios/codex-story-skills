# STARTER_RULES

## Типы starter pack

### Минимальный
Подходит для:
- новой идеи
- короткого романа
- проекта на ранней стадии

Создавать:
- `canon/current-canon.md`
- `canon/system.md`
- `characters/protagonist.md`
- `books/book-1/outline.md`
- `SKILL.md`
- `README.md`

### Стандартный
Подходит для:
- одного полноценного романа
- проекта, где уже есть герой, мир и крупный конфликт

Создавать:
- `canon/current-canon.md`
- `canon/system.md`
- `canon/conflict-matrix.md`
- `characters/protagonist.md`
- `characters/character-template.md`
- `books/book-1/outline.md`
- `books/book-1/chapter-map.md`
- `books/book-1/chapter-grid.md`
- `books/book-1/act-breakdown.md`
- `books/book-1/scenes/`
- `archive/`
- `inbox/`
- `prompts/task-template.md`
- `prompts/session-header.md`
- `SKILL.md`
- `README.md`

### Расширенный
Подходит для:
- цикла книг
- большого мира
- проекта с несколькими сюжетными линиями

Создавать:
- всё из стандартного starter pack
- `books/book-2/outline.md`
- `books/book-3/outline.md`
- `characters/supporting/`
- `world/`
- `world/locations.md`
- `world/factions.md`
- `world/history.md`

### Миграционный
Подходит для:
- проекта, где уже есть старая база
- заметок из Obsidian
- архива старых веток

Дополнительно создавать:
- `archive/old-notes/`
- `archive/discarded-branches/`
- `archive/imported-materials/`
- `inbox/`

И сразу рекомендовать:
- отделить рабочую базу от архива
- не считать старые материалы каноном
- сначала собрать `current-canon.md`

---

## Правила выбора

1. Не перегружать маленький проект структурой большого цикла.
2. Не создавать chapter-level документы до появления базового канона и outline.
3. Если у пользователя уже есть хаотичная база, использовать миграционный режим.
4. Если масштаб проекта неясен, предлагать стандартный starter pack как базовый вариант.
