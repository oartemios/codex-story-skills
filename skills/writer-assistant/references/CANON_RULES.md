# CANON_RULES

## Работа с каноном

### current-canon.md

Использовать как главный короткий снимок текущего рабочего канона.

Структура по умолчанию:
- статус проекта
- основа истории
- главный герой
- ключевые персонажи
- мир / системный слой
- книга 1
- следующие книги, если они уже зафиксированы
- исключенные ветки
- не зафиксировано

## Связанный шаблон

Использовать:

- ../templates/current-canon-template.md

Шаблон использовать как опорную форму.
Если проекту не нужен отдельный блок, его можно сократить, но нельзя убирать:
- статус
- подтвержденный канон
- исключенные ветки
- не зафиксировано

### Правила

- Канон фиксируется только на основе подтвержденных решений
- Противоречия не сглаживать — фиксировать явно
- Любая новая идея сначала проходит как рабочая версия
- `current-canon.md` не должен превращаться в склад идей, синопсисов и альтернативных версий

### Разделение

Всегда различать:
- Канон
- Рабочая версия
- Идеи на будущее

### Обновление канона

При изменении:
1. Проверить конфликт с текущим каноном
2. Если конфликт есть — явно указать, что именно конфликтует
3. Определить решение:
   - подтверждаем новую версию
   - сохраняем старую версию
   - выносим вопрос в `Не зафиксировано`
4. Только после решения обновлять канон
5. Если старая версия снята, перенести ее в `Исключенные ветки`, а не удалять бесследно

### Исключенные ветки

Фиксировать, если:
- ветка раньше рассматривалась как рабочая
- она конфликтует с текущим решением
- есть риск ее случайного возврата

Для каждой исключенной ветки кратко фиксировать:
- что именно исключено
- на каком уровне
- чем заменено или почему снято

### Не зафиксировано

Использовать, если:
- нет данных
- есть противоречия
- есть несколько вариантов
