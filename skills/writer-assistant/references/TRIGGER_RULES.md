# TRIGGER_RULES

## Правила подключения модулей

Сначала определить тип задачи.
Потом подключать только те модули, без которых нельзя дать рабочий результат.
Если задача относится к нескольким уровням сразу, подключать несколько модулей, но не дублировать между ними одни и те же требования.

### STARTER_RULES.md
Использовать, если задача связана с:
- starter pack
- новой структурой проекта
- запуском нового проекта
- шаблоном репозитория
- миграцией в новую структуру

### CANON_RULES.md
Использовать, если задача связана с:
- current-canon.md
- фиксацией канона
- проверкой противоречий
- разделением канона и рабочих гипотез
- исключенными ветками
- обновлением рабочего канона после решений

Шаблон подключать, если нужен рабочий `current-canon.md`:
- ../templates/current-canon-template.md

### SYSTEM_RULES.md
Использовать, если задача связана с:
- system.md
- устройством мира
- правилами магии / технологии
- ограничениями и ценой системы
- разделением системного слоя и сюжетных следствий

Шаблон подключать, если нужен рабочий `system.md`:
- ../templates/system-template.md

### CONFLICT_RULES.md
Использовать, если задача связана с:
- conflict-matrix.md
- конфликтными узлами
- напряжением между персонажами
- скрытыми удерживающими факторами
- проверкой, держат ли отношения драматическое давление

Шаблон подключать, если нужна конфликтная матрица:
- ../templates/conflict-matrix-template.md

### CHARACTER_RULES.md
Использовать, если задача связана с:
- карточками персонажей
- проработкой героя
- функцией персонажа
- аркой персонажа
- отношениями персонажей

Шаблон подключать, если нужна полноценная карточка:
- ../templates/character-template.md

### STRUCTURE_RULES.md
Использовать, если задача связана с:
- outline
- chapter-map
- chapter-grid
- act-breakdown
- структурой книги
- количеством глав
- распределением функций глав
- переходом между уровнями структуры

Шаблоны подключать по уровню задачи:
- outline -> ../templates/outline-template.md
- chapter-map -> ../templates/chapter-map-template.md
- act-breakdown -> ../templates/act-breakdown-template.md

`chapter-grid` использовать без отдельного шаблона, если задача не требует специального формата.

### SCENE_RULES.md
Использовать, если задача связана с:
- scene-grid
- scene-plan
- разбором главы по сценам
- функцией сцены
- ритмом эпизодов

Подключать только после устойчивой chapter-level структуры.

Шаблон подключать, если нужен scene plan:
- ../templates/scene-plan-template.md

### ARCHIVE_RULES.md
Использовать, если задача связана с:
- архивом
- inbox
- переносом старой базы
- старыми заметками
- исключенными ветками

## Типовые сочетания модулей

- `CANON_RULES.md` + `SYSTEM_RULES.md`: когда меняется канон и системный слой одновременно
- `CANON_RULES.md` + `CONFLICT_RULES.md`: когда конфликтная матрица влияет на подтвержденный канон
- `STRUCTURE_RULES.md` + `SCENE_RULES.md`: когда задача идет от структуры главы к сценам
- `CHARACTER_RULES.md` + `CONFLICT_RULES.md`: когда нужно собрать персонажа через его давление на других
- `ARCHIVE_RULES.md` + любой доменный модуль: когда старый материал нужно сначала проверить на совместимость, а потом уже перерабатывать

## Проверка перед ответом

- Понять, к какому уровню относится задача
- Не подключать scene-level раньше устойчивой структуры
- Не подключать шаблон без доменного модуля
- Если шаблон уже есть, усиливать связанный модуль, а не создавать новый без необходимости
