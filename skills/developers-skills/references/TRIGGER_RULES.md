# TRIGGER_RULES

## RFC

Выбирать RFC, если задача про:

- proposal
- design doc
- планируемое техническое изменение
- обсуждение еще не принятого решения
- выбор между вариантами
- migration или rollout
- сбор обратной связи до реализации

Типовые слова: `RFC`, `proposal`, `design doc`, `предложение`, `спроектируй`, `варианты`, `rollout`, `migration`, `non-goals`.

## ADR

Выбирать ADR, если задача про:

- уже принятое решение
- запись архитектурного решения
- фиксацию причины выбора
- supersede или reject старого решения
- краткую историческую запись для будущих maintainers

Типовые слова: `ADR`, `decision record`, `architecture decision`, `зафиксируй решение`, `accepted`, `rejected`, `superseded`.

## Review без создания документа

Выбирать review-режим, если пользователь просит:

- проверить существующий RFC/ADR
- найти слабые места
- оценить последствия
- привести документ к форме

В review-режиме не переписывать весь документ, если достаточно точечных правок или списка замечаний.
