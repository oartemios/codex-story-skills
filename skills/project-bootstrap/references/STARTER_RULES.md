# STARTER_RULES

## Назначение

Создание нового проекта.

---

## Правила

- создавать только необходимый минимум
- использовать стандартную структуру
- не создавать лишние книги и файлы
- все создаваемые по шаблону рабочие документы должны содержать блок метаданных с версией
- начальная версия: 0.1
- сначала определить достаточный starter scope
- различать обязательное ядро и опциональные стартовые документы
- не создавать файлы "на вырост", если у них нет рабочей функции на старте

---

## Starter scope

Перед выдачей starter pack определить:

- проект создаётся с нуля или достраивается
- нужен ли только каркас папок или сразу опорные документы
- есть ли уже зафиксированный герой, канон, книга, README

### Обязательное ядро

- структура папок проекта
- базовые канонические опоры
- один минимальный вход в книжный слой

### Опционально

Добавлять только если есть явная функция:

- `README.md`, если нужен внешний обзор проекта
- `characters/protagonist.md`, если уже есть центральный герой
- `books/book-1/outline.md`, если книга уже мыслится как рабочая единица

---

## Критерии достаточности

Starter pack достаточен, если:

- проект можно продолжать без немедленного создания новых служебных файлов
- источники истины распознаны
- не возникает дублирования между каноном, персонажами и книгами
- стартовая структура не заставляет заранее создавать лишние книги, акты или персонажей
