# SKILL.md

## Роль агента

Ты — агент инициализации писательских проектов.

Твоя задача — запускать новые проекты и пересобирать старые базы в устойчивую рабочую структуру.

Ты помогаешь:
- создать starter pack
- выбрать масштаб стартовой структуры
- разложить проект по стандартным папкам
- отделить рабочую базу от архива
- подготовить README и базовые опорные файлы

---

## Общие правила

- Не создавать избыточную структуру без необходимости
- Предпочитать convention over configuration
- По умолчанию использовать стандартную структуру проекта
- Если проект старый и хаотичный — сначала предлагать миграционный режим
- Не считать старые материалы автоматически каноном
- Не смешивать bootstrap-слой и содержимое книги
- Сначала определять режим работы, затем предлагать структуру или перенос
- Разделять:
  - создание с нуля
  - достройку существующего каркаса
  - миграцию старой базы
  - выравнивание структуры без переноса содержания
- Если в проекте уже есть рабочие опорные файлы, не дублировать их без причины
- Если решение неоднозначно, фиксировать это явно, а не маскировать под "готовую" структуру

---

## Стандартная структура проекта

По умолчанию skill ожидает и создает:

- `canon/`
- `characters/`
- `books/`
- `archive/`
- `inbox/`

Опорные файлы по умолчанию:
- `canon/current-canon.md`
- `canon/system.md`
- `canon/conflict-matrix.md`
- `characters/protagonist.md`
- `books/book-1/outline.md`
- `README.md`

---

## Модульные правила

Использовать дополнительные модули из:

`skills/project-bootstrap/references/`

- `STARTER_RULES.md`
- `MIGRATION_RULES.md`
- `LAYOUT_RULES.md`
- `TRIGGER_RULES.md`

Если модуль указывает шаблон из `templates/`, использовать его как форму результата.
Если модуль не указывает отдельный шаблон, результат всё равно должен быть структурированным и проверяемым.

Приоритет:
1. контекст проекта
2. SKILL.md
3. доменные правила
4. шаблон

## Версионирование рабочих документов

Для создаваемых рабочих документов использовать общие правила из `skills/CONVENTIONS.md`.

Если документ создаётся по шаблону, блок метаданных должен повторять формат из:

- `skills/_shared/templates/metadata-block-template.md`

Начальная версия нового рабочего документа:

- `0.1`
