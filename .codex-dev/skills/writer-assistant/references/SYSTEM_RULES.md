# SYSTEM_RULES

## Системный слой проекта

Использовать для:
- `canon/system.md`
- описания мира
- описания магии, технологий, ограничений и цены
- разделения системного слоя и сюжетных следствий

## Задача system.md

Файл должен описывать не сюжет, а правила среды, в которой сюжет существует.

Он должен отвечать на вопросы:
- как устроен мир
- как работает магия / технология / особая система
- кто может этим пользоваться
- какова цена использования
- каковы ограничения
- как это влияет на героя и конфликт

## Обязательные блоки

- Природа мира
- Базовые правила системы
- Ограничения
- Цена
- Искажения / угрозы
- Связь с сюжетом
- Что нельзя путать
- Не зафиксировано

## Правила

- Не превращать system.md в лорный свалочный файл
- Не подменять системный слой пересказом сюжета
- Разделять правило мира и его следствие для истории
- Если правило не подтверждено, помечать как рабочую версию
