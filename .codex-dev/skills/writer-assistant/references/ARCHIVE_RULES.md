# ARCHIVE_RULES

## Разделение зон

`archive/` и `inbox/` не являются каноном.

### archive/

Использовать для:
- старых версий материалов
- исключенных веток
- импортированных заметок
- документов, которые не участвуют в текущем рабочем контуре

### inbox/

Использовать для:
- новых несортированных входящих материалов
- идей без статуса
- заметок, которые еще не прошли проверку на совместимость

`archive/` хранит уже отведенный в сторону материал.
`inbox/` хранит материал, который еще не разобран.

## Правила

- архив не является скрытым источником истины
- старые материалы не считать автоматически истинными
- не смешивать архив, inbox и рабочую базу
- использовать старые материалы только при явном указании задачи
- сначала проверять совместимость с текущей рабочей базой, потом решать судьбу материала

## Работа со старыми материалами

При разборе старого файла или ветки:

1. Определить тип материала:
   - старый канон
   - черновик
   - исключенная ветка
   - неразобранная идея
2. Проверить совместимость с текущей рабочей базой.
3. Явно зафиксировать решение по материалу.
4. Не переносить материал в рабочий контур автоматически.

## Логика решений

### Сохранить как архив

Использовать, если материал:
- важен для истории решений
- нужен как след старой версии
- не должен влиять на текущий канон

### Переработать

Использовать, если материал:
- частично совместим с текущей базой
- содержит рабочее зерно, но в старой форме
- требует пересборки, а не прямого переноса

### Вынести в inbox

Использовать, если материал:
- потенциально полезен
- еще не проверен
- требует отдельного решения позже

### Исключить из рабочего контура

Использовать, если материал:
- противоречит текущему канону
- возвращает исключенную ветку
- дублирует уже снятое решение
- не нужен ни как архив, ни как входящий материал

## Миграция

При переносе старого проекта:
- всё старое → archive/
- всё неразобранное входящее → inbox/
- рабочая база собирается заново
- канон фиксируется отдельно
