# ADR_RULES

## Назначение

ADR фиксирует принятое или отклоненное архитектурное решение. Его цель — сохранить контекст для будущих maintainers.

## Обязательные блоки

- Статус
- Контекст
- Решение
- Рассмотренные альтернативы
- Последствия
- Follow-ups

## Правила

- Status должен быть явным: `Proposed`, `Accepted`, `Rejected`, `Deprecated` или `Superseded`.
- Если решение еще не принято, не писать `Accepted`.
- Context описывает силы и ограничения, которые сделали решение нужным.
- Decision должен быть коротким и проверяемым.
- Alternatives должны объяснять, почему варианты не выбраны.
- Consequences включают positive, negative и neutral последствия.
- Если ADR заменяет старый документ, указать superseded/supersedes связь.
